## Problem

Claude Desktop App und Claude Code (VS Code) sind komplett getrennte Instanzen. Kein geteilter Kontext, kein Memory, keine Chat-History. Wer beides nutzt, wird zum menschlichen Router.

## Loesung

VS Code Extension die Claude Code JSONL-Sessions automatisch in lesbares Markdown konvertiert. One-Click oder Auto-Sync, kein Terminal noetig.

## Entscheidungen

| Frage | Entscheidung | Begruendung |
|-------|-------------|-------------|
| Repo-Struktur | Monorepo (`cli/` + `extension/`) | Ein Repo, ein Brand, einfacher zu maintainen |
| Architektur | Pure TypeScript | Keine Python-Dependency fuer User, niedrigste Einstiegshuerde |
| Monetarisierung | 100% Free Launch, Freemium spaeter | Community aufbauen, dann monetarisieren |
| MVP-Scope | Statusbar + Auto-Sync | KISS — Quality over Features |
| Sidebar Panel | NICHT in v1.5 | Overengineering, spaeter als v1.6+ Feature |
| Multi-KI | NICHT in v1.5 | Roadmap v2.0 |

## Core Features (v1.5 MVP)

### 1. Statusbar Button
- Zeigt Sync-Status: "Schroedinger: 3 new" oder "Schroedinger: synced"
- Klick triggert manuellen Sync
- Icon: Katze oder Sync-Symbol

### 2. Auto-Sync
- FileSystemWatcher auf `~/.claude/projects/` (alle `*.jsonl`)
- Debounced (wartet 5 Sekunden nach letzter Aenderung)
- Konfigurierbar: an/aus

### 3. JSONL Parser (TypeScript-Port)
- Liest Claude Code JSONL-Sessions
- Extrahiert: Metadata, User-Messages, Assistant-Messages, Tool-Usage
- Identisch zur Python-Logik, portiert nach TypeScript

### 4. Markdown Generator
- Generiert saubere Markdown-Summaries pro Session
- Format: Datum, Session-ID, Projekt, Branch, Turns, Tool-Statistik, Gespraechsverlauf
- Output: `./sync/` Ordner (konfigurierbar)

### 5. State Tracking
- JSON-State-File trackt bereits synchronisierte Sessions
- Hash-basierte Change-Detection (Groesse + Modification Time)
- Ueberspringt bereits verarbeitete, unveraenderte Sessions

## Settings (3 Stueck, KISS)

{

  "schroedingerSync.outputDir": "./sync",
  "schroedingerSync.gitAutoCommit": false
}
