## 1. Produktvision

### Das Problem
Jede KI-Instanz (Claude, Gemini, DeepSeek, ChatGPT) startet bei Null. Kein geteiltes Gedaechtnis, keine Faktenbank, keine Verifikation. User muessen Kontext manuell uebertragen — oder riskieren dass eine KI Muell erzaehlt den eine andere laengst korrigiert hat.

### Die Erkenntnis
Michaels manuelles System (Memory-Files, Rules, Korrekturen-Tabellen, Topic-Files) IST bereits ein funktionierender Sync-Prototyp. v2.0 automatisiert genau das.

### Die Loesung
VS Code Extension die aus KI-Sessions automatisch strukturiertes Wissen extrahiert, verifiziert und KI-uebergreifend bereitstellt. Zwei Schichten:

| Schicht | Tier | Was der User sieht |
|---------|------|---------------------|
| **Memory Layer** | Free | Auto-generierte Topic-Files (.md), Export-Button fuer andere KIs |
| **Knowledge Layer** | Pro | Graph-basierte Faktenbank, Widerspruchs-Erkennung, Cross-KI Verifikation |

### Taglines
- "Your AI instances finally share a brain."
- "One memory. Every AI. Zero bullshit."
- "Is your AI synchronized? Yes. And no. — until you install Schroedinger Sync."

### Evolution
- v1.0 — Session-Dumper (Python CLI) ✓
- v1.5 — Session-Dumper (VS Code Extension) ✓ (Task 8 pending)
- **v2.0 — AI Knowledge Graph** (dieses Dokument)

## 2. Entscheidungen

| Frage | Entscheidung | Begruendung |
|-------|-------------|-------------|
| Zielgruppe | Breiter Markt + Enterprise | Maximales Potenzial, Firmen zahlen Premium |
| Scope | Full Package (Fakten, Korrekturen, Entscheidungen, Tasks, Rules) | Alles oder nichts — es funktioniert oder nicht. Schroedinger Style. |
| Storage | Lokal + Clipboard/Export | Privacy-first, DSGVO by design, kein Server = keine Kosten |
| Extraktion | Hybrid (Regelbasiert Free, LLM-gestuetzt Pro) | Freemium-Hebel: 80% gratis, 99% fuer Bezahler |
| Architektur | Multi-File Memory (Free) + Knowledge Graph (Pro) | Bewaehrt durch Michaels Prototyp + skalierbar fuer Enterprise |
| Lizenz | Open Core (Free=MIT, Pro/Team/Enterprise=Proprietary) | GitLab/Supabase-Modell: Community anlocken, Premium monetarisieren |

interface StorageBackend {

### Datenmodell (Knowledge Graph Node)

```typescript
interface Fact {
  id: string;
  category: "person" | "date" | "decision" | "correction" | "task" | "rule";
  subject: string;        // "Eva Popow"
  predicate: string;      // "hatGdB"
  value: string;          // "30"
  confidence: "verified" | "extracted" | "conflict";
  sources: string[];      // ["claude:session-abc", "gemini:export-123"]
  createdAt: string;
  updatedAt: string;
  status: "active" | "archived";  // Time-Decay / Relevanz
  supersedes?: string;    // ID des alten Fakts bei Korrektur
}

interface Conflict {
  factA: string;          // Fact ID
  factB: string;          // Fact ID
  field: string;          // "value"
  valueA: string;         // "20"
  valueB: string;         // "30"
  sourceA: string;        // "chatgpt:session-xyz"
  sourceB: string;        // "claude:session-abc"
  resolvedBy?: string;    // User ID
  resolvedAt?: string;
  resolution?: string;    // Fact ID des verifizierten Fakts
}
```

## 4. Security & Verschluesselung

### Bedrohungsszenarien

| Angriff | Risiko | Loesung |
|---------|--------|---------|
| Malware liest lokale Files | HOCH | Encryption at Rest (Pro) |
| Man-in-the-Middle bei Export | MITTEL | Encrypted Export + 60s Clipboard TTL |
| Fremder Zugriff auf PC | MITTEL | Master-Passwort via OS Keyring |
| LLM-API leakt Daten | NIEDRIG | On-Prem LLM Option (Enterprise) |
| Backup auf USB/Cloud geklaut | HOCH | Encrypted Container |

### Verschluesselungs-Stufen

| Tier | Schutz |
|------|--------|
| Free | Klartext + .gitignore fuer sensitive Files |
| Pro | AES-256 Encryption at Rest, Master-Key via OS Keyring (Win: Credential Manager, macOS: Keychain, Linux: libsecret), Dateien verschluesselt auf Disk, entschluesselt nur im RAM |
| Team | Pro + Key-Sharing via asymmetrische Verschluesselung (jeder User eigener Key) |
| Enterprise | Pro + Team + HSM/TPM Support, Audit-Log, Key Rotation Policy, Compliance-Report Generator |

### Security Provider Interface

```typescript
interface SecurityProvider {
  encrypt(data: string): Promise<Buffer>;
  decrypt(data: Buffer): Promise<string>;
  isLocked(): boolean;
  unlock(credential: string): Promise<void>;
  lock(): void;
}

// Free: PlaintextProvider (passthrough)
// Pro: AESProvider (OS Keyring fuer Master-Key)
// Enterprise: HSMProvider (Hardware Security Module)
```

### Export-Sicherheit
- Clipboard wird nach 60 Sekunden automatisch geloescht (wie Passwort-Manager)
- Verschluesselter Export mit Einmal-Passwort (Pro)
- RAM wird gewiped wenn VS Code schliesst oder User sperrt

### Zertifizierungs-Roadmap

| Wann | Was | Warum |
|------|-----|-------|
| v2.0 | OS Keyring + AES-256 | Basis-Schutz |
| v3.0 | SOC 2 Type II | Enterprise USA |
| v3.0 | DSGVO-Konformitaetserklaerung | EU-Pflicht |
| v3.x | ISO 27001 | Grosskonzerne |

## 9. Confidence & Time-Decay

### Confidence-Level pro Fakt

| Level | Bedeutung | Wer setzt es |
|-------|-----------|-------------|
| **verified** | User hat bestaetigt | User-Klick oder Pro Conflict Resolution |
| **extracted** | Automatisch aus Session | Extractor (Free/Pro) |
| **conflict** | Zwei Quellen widersprechen sich | Conflict Detector (Pro) |

### Time-Decay / Relevanz
- Fakten haben Status `active` oder `archived`
- Tasks die erledigt sind werden automatisch archiviert
- Archivierte Fakten erscheinen NICHT im Export (spart Tokens)
- User kann manuell archivieren/reaktivieren
- Pro: Auto-Archivierung nach konfigurierbarer Zeit ohne Referenz
